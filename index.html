第一章：关于this

为什么要用this

this的主要功能是提高代码复用性，在很多情况下可以隐式”传递”一个对象引用。随着代码的越来越复杂，显式传递上下文对象会让代码变得混乱。

关于this的一些误解

1. this是指向自身的

当执行foo.count = 0时，的确向foo函数对象添加了一个count属性。但是函数代码中this.count中的this并不是指向那个函数对象。
虽然属性名相同，根对象却不相同。这段代码其实是无意中创建了一个全局变量count，它的值是Nan。

2. this的作用域指向词法作用域

this在任何情况下都不指向函数的词法作用域，作用域和对象类似，里面的可见标识符都是它的属性，但是作用域”对象”无法通过JavaScript代码访问，因为存在在引擎内部。
this和词法作用域的查找无法混合使用。

this是什么

this既不指向自身，也不指向函数的词法作用域。this实际上是在函数被调用时发生的绑定，它指向什么完全取决于函数在哪里被调用。当函数被调用时，会创造一个活动记录（执行上下文）。
这个记录包括了函数在哪里被调用（调用栈）、函数的调用方法、传入参数等。this就是其中的一个属性。

第二章：this全面解析

调用位置

判断调用位置最重要的是分析调用栈（为了达到执行位置所执行的所有函数）。我们所关心的调用位置（也就是this绑定到的位置）就在当前执行函数的前一个调用中。

绑定规则

1. 默认绑定

独立调用函数就是默认绑定，是无法使用其他规则的时候的默认规则。

  • 非严格模式：this绑定在全局对象上
  • 严格模式：this绑定到undefined

虽然this的绑定规则完全取决于调用位置，但是这仅仅适用于包含this的函数要在non-strict模式，默认绑定才能绑定全局对象；use strict模式下与函数调用位置无关。

JavaScript中的new和其他语言不同，JavaScript中使用new并不会构造一个类，也不会实例化一个类，它只是一种调用函数的方式而已。

  • 这个新对象会绑定到函数调用的this

  1. 函数是否在new中调用(new绑定)？如果是的话this绑定的是新创建的对象。

    var bar = obj1.foo()

  4. 如果都不是的话，使用默认绑定。如果在严格模式下，就绑定到undefined，否则绑定到 全局对象。

    var bar = foo()

绑定规则的例外情况

  1. 如果把null或者undefined作为this的绑定对象传入call、apply或者bind，这些值在调用的时候会被忽略，实际使用的是默认绑定。

第三章：对象

————函数调用位置的不同会导致this绑定对象的不同，那什么是对象，为什么要绑定它们呢？

语法

对象可以通过两种方式定义：

  • 文字形式：var myObj = { key:value, ... }
  • 构造形式：var myObj = new Object(); myObj.key = value

类型

简单基本类型：string, boolean, number, null, undefined。简单基本类型本身并不是对象

内置对象：String, Number, Boolean, Object, Function, Array, Date, RegExp, Error。
但实际上在JavaScript中，它们只是一些内置函数，它们可以当作构造函数（new调用的函数）来使用，从而可以构造一个对应子类型的新对象。

JavaScript中字面量和对象可以互相转换，比如对于一个普通的string使用.length时，会自动”装箱”，并不需要显式创建new String()对象。

注意：null和undefined没有对应的构造形式，只有文字形式；Date没有文字形式。

内容

当说起”内容”的时候，虽然看起来这些值看起来被储存在对象的内部，但实际上储存在内部的是这些名称，它们就像指针（引用），指向真正的位置。

访问对象内部的值的时候，可以通过.（属性访问）或者[]（键访问）。在对象中，属性名永远是字符串，如果使用的不是string的值，也会被改成string。
如果传入的是对象，例如obj[myObject]，也会变成obj["[Object Object]"]

在JavaScript中函数和方法没有区别，即使在对象中文字声明了一个函数表达式，这个函数也不会属于这个对象，它们只是对于相同函数对象的多个引用。例如：

数组

数组也是一个对象，如果试图向数组添加一个属性，如果属性名是一个数字（或者字符串型的数字），那么实际上是给他添加了一个数值下标。（他会修改数组的内容而不是添加一个属性）。

如果数组（var myArray = [ "foo", 42, "bar" ];）添加了一个属性(myArray.baz = "baz";)，则并不会改变数组本身，而是添加了一个属性（baz）。

对象复制

  • 浅拷贝：Object.assign(...)，第一个参数是目标对象，后面是源对象，他会遍历多个源对象所有可枚举(enumerable)的自有键(owned key)，并把它们用=复制。
  • 深拷贝：var newObj = JSON.parse( JSON.stringify(someObj) );

属性描述符

  • writable：决定属性是否可修改
  • configurable：如果是true，说明可以用defineProperty(...)的方法来修改属性符。
  • enumerable：是否可以被枚举，也就是是否可以用for循环遍历。

不变性

所有的方法创建都是浅不变形：也就是说只能保证对象自己不变，不能保证对象的属性（如果引用了其他对象）也不变。如果想实现不变形：

  1. 对象常量：设置writable和configurable为false即可
  2. 禁止扩展：可以用Object.preventExtension(myObject)
  3. 密封：Object.seal(myObject)创建一个”封闭”的对象，会在调用preventExtension的时候configurable:false
  4. 冻结：Object.freeze(myObject)在seal的基础上设置writable:false

最小暴露原则：在软件设计中，应当最小限度地暴露必要内容，而将其他内容隐藏起来。如定义内部函数等。

  • 好处：避免同名标识符之间的冲突。 此外，通过函数代码块的形式来包装函数，会产生额外的问题：1）必须声明一个具名函数，意味着这个函数本身
    污染了所在作用域（即在作用域中添加了或许不必要的标识符）。2）必须显式地通过函数名来调用这个函数。

  • 匿名表达式：没有函数标识符的表达式

第五章：作用域闭包

闭包的定义理解

闭包：当函数可以记住并访问所在的词法作用域（气泡）时，即使函数是在当前词法作用域之外执行。

函数bar()的词法作用域能够访问foo()的内部作用域，然后将bar当作一个值进行传递。
foo()执行时，实际上是调用了bar()。在这里，bar()在自己定义的词法作用域以外的地方执行。
当foo()执行后，由于闭包的存在，foo的内部作用域实际上是依然存在的，因此不会被回收，以供bar()在之后任何时间进行引用。

把内部函数baz()传给bar()，baz()能够访问foo()的作用域。而在bar(baz)调用时，在bar()作用域内部的fn()实际上访问了foo()的作用域。

本质上无论何时何地，如果将函数（访问它们各自的词法作用域）当作第一级的值类型并到处传递，就会看到闭包的应用。（例如回调函数，定时器等）

模块模式的两个必备条件：

  • 必须有外部封闭函数，该函数至少被调用一次（每次调用都会创建一个新的实例）。
  • 封闭函数必须返回至少一个内部函数，这样内部函数才能在私有作用域中形成闭包，并且可以访问或者修改私有状态。 闭包也是一个非常强大的工具，可以用多种形式来实现模块等模式，如上述例子中，通过调用foo.XXX()的方式实现了在其他作用域中使用foo作用域内的变量。

动态作用域

JavaScript语言使用词法作用域，即写代码的时候就已经确定了作用域，它更关注在何处声明；动态作用域是在运行时确定的（this也是），它更关注从何处调用。

动态作用域会顺着调用栈在调用函数foo()的地方查找a，而不是嵌套词法作用域链向上查找。

this词法
